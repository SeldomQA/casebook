## 0.6.0

- 新增：测试计划支持 `Full run` 和 `Retest failed/blocked/deferred` 两种执行模式。
- 新增：支持基于上一轮失败、阻塞、延期用例创建下一轮重测计划。
- 新增：执行结果增加 `Deferred`，用于标记需要延后到下一轮验证的用例。
- 优化：测试计划未全部执行时，不允许完成计划。
- 优化：测试计划创建区域区分 `Full run` 和 `Retest`，来源计划只在重测模式下显示。
- 优化：测试计划详情中，执行信息与评审标记互斥展示。
- 优化：报告按测试计划实际范围统计，支持重测轮次结果展示。

## 0.5.0

- 新增：`casebook export` 命令，支持将 YAML 用例导出为独立 HTML 文件。
- 新增：导出 HTML 支持目录聚合、单文件导出、`--tag` 和 `--priority` 过滤。
- 新增：静态 HTML 支持评审标记、备注和本地保存，方便会议评审和冒烟用例分发。
- 优化：`casebook export` 与 `casebook serve` 的用例列表、展开样式、标签样式保持一致。
- 优化：统一 Mark 交互，标记后展示标签，不再整行变色。
- 优化：用例详情中的 Preconditions、Steps、Expected Results 样式与本地工作台保持一致。

## 0.4.0

- 新增：`casebook renumber` 命令，支持按当前 YAML 文件顺序重新整理用例 ID。
- 新增：页面增加 `ID Update` 能力，用例评审后可快速修正编号顺序。
- 新增：测试计划模式支持按执行结果筛选用例。
- 新增：执行详情恢复 `Defects` 字段，支持记录 JIRA 或缺陷链接。
- 优化：命令行输出引入字符画 Banner，展示版本、访问地址、范围、用例数量等信息。
- 优化：引入 `loguru` 统一日志输出。
- 优化：项目中的用户提示统一为英文。

## 0.3.0

- 新增：`casebook report <run-file>` 命令，支持生成 HTML 测试报告。
- 新增：报告包含测试计划信息、执行概览、失败用例和阻塞用例。
- 新增：报告引入图表统计，展示通过率和执行状态分布。
- 新增：`casebook serve` 页面 favicon。
- 优化：测试计划信息展示，支持环境、测试人员、完成时间等基础信息。
- 优化：完成测试计划时自动写入完成时间，便于生成报告。

## 0.2.0

- 新增：`casebook init` 项目脚手架，生成 Schema、示例用例、技能包和编辑器配置。
- 新增：引入 AI Agent 技能包，指导 AI 根据需求生成符合 Schema 的 YAML 用例。
- 新增：CLI 改为 Typer 实现，命令结构更清晰。
- 新增：测试计划最小可用能力，支持创建、选择和保存执行轮次。
- 新增：`test-runs/*.json` 存储执行结果。
- 新增：用例支持 `Passed`、`Failed`、`Blocked` 执行状态。
- 新增：顶部进度条和执行统计卡片。

## 0.1.0

- 新增：首个可用的 `casebook serve` 本地工作台。
- 新增：扫描 `releases/` 下的 YAML 测试用例并展示文件树。
- 新增：用例列表、详情展开、优先级、类型、标签展示。
- 新增：用例 Mark 标记能力，支持在评审中记录需要更新的用例。
- 新增：轻量编辑能力，可在页面中修改用例内容并写回 YAML。
- 新增：文件监听能力，YAML 变更后页面可自动刷新。
